♦️ 分支策略 ​

分支命名 ​

  • main - 主开发分支
  • doc - 文档分支
  • feat/* - 功能分支
  • fix/* - 修复分支
  • refactor/* - 重构分支

分支保护规则 ​

  • ✅ 需要 PR review
  • ✅ 必须通过 CI 检查
  • ✅ 代码检查必须通过
